# Schedules — Sales Call Demo Script

**Product:** Letta Schedules  
**Tagline:** *Set it now, get it back at the right time.*

## Demo goal
Show that Schedules turns a one-time request into a dependable follow-up workflow that returns later with the original context.

## Pre-call setup
- Open Letta Desktop
- Have the **Schedules** tab ready
- Keep the dashboard page open in a browser tab for proof-of-work if needed
- Have one simple example ready:
  - Title: `Follow up with client`
  - Message: `Check back tomorrow at 9 AM and confirm the proposal was sent.`
  - Timing: near-future or recurring daily

## Step-by-step demo script

### 1) Open with the business problem
**Talk track:**
> “Most teams do not lose deals because they lack reminders. They lose deals because follow-ups slip, recurring checks are manual, and the context gets lost between tools.”

**What to do:**
- Ask the customer how they currently manage reminders, check-ins, renewals, and status nudges.
- Listen for missed follow-ups, repetitive manual work, and tool sprawl.

### 2) Position Schedules as the fix
**Talk track:**
> “Schedules lets Letta hold a task, reminder, or recurring check and bring it back later automatically.”

**What to do:**
- Show the Schedules tab.
- Point out that this is a native workflow, not a separate automation product.

### 3) Create a schedule live
**Talk track:**
> “I’ll create a simple follow-up now, and Letta will bring it back at the right time with the original context intact.”

**What to do:**
- Create a schedule with a clear title and message.
- Set it for a near-future time or a recurring slot.
- Save it.

### 4) Confirm the schedule is active
**Talk track:**
> “The important part is that this is durable — it is stored, visible, and ready to trigger later.”

**What to do:**
- Show the new schedule in the active list.
- Highlight recurrence, timing, and message fields.

### 5) Show how it behaves over time
**Talk track:**
> “This is where Schedules becomes valuable: Letta doesn’t just remember the request, it returns with the right action at the right time.”

**What to do:**
- If available, switch to the dashboard to show recent runs and evidence.
- Mention that schedules can feed recurring briefings, inbox triage, renewals, and status follow-ups.

### 6) Tie it to the customer’s workflow
**Talk track:**
> “The best use cases are anything that should happen again without someone having to remember it manually.”

**What to do:**
- Map the demo to their process:
  - customer follow-up
  - account review
  - project check-in
  - weekly status summary
  - renewal reminder

### 7) Close with a business outcome
**Talk track:**
> “The win is not just automation. It is fewer missed follow-ups, less manual tracking, and a more proactive assistant.”

**What to do:**
- Summarize value in one sentence.
- Ask for agreement on a pilot use case.

## Key talking points

### 3 benefits
1. **Fewer missed follow-ups** — schedules keep important actions from falling through the cracks.
2. **Less manual tracking** — recurring reminders and check-ins no longer depend on someone remembering them.
3. **More proactive assistant behavior** — Letta returns later with context, so the assistant feels dependable instead of reactive.

### 3 differentiators
1. **Native to Letta** — schedules live inside the assistant experience, not in a separate automation silo.
2. **Context-aware follow-up** — the saved task comes back with the same meaning and business context.
3. **Local-first and demo-friendly** — the feature can be shown without depending on a complex external stack.

## Objection handling

### 1) “We already use Zapier.”
**Response:**
That is fine for app-to-app automation, but Schedules is built into Letta, so the assistant itself owns the follow-up. You get context, continuity, and a native workflow instead of stitching another tool onto the stack.

### 2) “This seems like simple reminders.”
**Response:**
The value is not just a reminder. It is a task that returns later with the original context and can support recurring follow-ups, briefings, and workflow handoffs.

### 3) “Our team will not adopt another tool.”
**Response:**
They do not need a new standalone product. Schedules sits in the Letta experience they already use, so the learning curve is low and the behavior is consistent.

### 4) “How secure is it?”
**Response:**
The local demo path is designed to be simple and controllable, and the surrounding implementation can run hidden or windowless when needed. For production, we would align permissions, retention, and audit controls to your policy.

### 5) “What is the ROI?”
**Response:**
If Schedules prevents even a few missed follow-ups or saves a few hours of manual tracking each week, it pays for itself quickly. The strongest ROI usually comes from sales follow-up, renewals, recurring status checks, and ops reminders.
